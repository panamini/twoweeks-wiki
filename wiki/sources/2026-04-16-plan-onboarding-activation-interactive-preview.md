## Résumé

Plan d'exécution pour la suite du cycle 1 produit après le parser cleanup. La note recommande de faire d'abord l'interactive preview ↔ editor linking, puis le quick-start guided path, puis le first-run onboarding system. Elle considère la brand voice comme suffisamment mature pour être appliquée directement sans chantier préalable.

## Points clés

1. L'interactive preview ↔ editor linking est priorisé comme premier wow moment et comme meilleur signal premium/retention.
2. Le quick-start guided path vient ensuite pour résoudre le blank-slate conversion problem.
3. Le first-run onboarding doit rester léger : checklist, nudges contextuels, spotlights, empty-state education, pas de tooltip tour.
4. La brand voice est jugée complète; elle doit être appliquée directement aux microcopies onboarding.

## Implications pour twoweeks

- La roadmap doit expliciter l'ordre d'exécution réel à l'intérieur du bloc onboarding/preview.
- Le produit doit traiter le preview comme surface d'entrée, pas comme simple chrome de sortie.

## Extraits notables

- "The interactive preview ↔ editor linking should come first."
- "Brand voice: don't touch it first. It is already solid."
