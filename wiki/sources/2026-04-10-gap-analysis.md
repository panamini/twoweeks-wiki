# Gap Analysis — Path C

**Type** : Document de diagnostic produit
**Date** : 2026-04-10

## Résumé

Troisième partie de la trilogie stratégique (benchmark matrix → success blueprint → gap analysis). Diagnostic brutal de ce qui empêche TwoWeeks de battre ResumeLab commercialement : 6 points qui nuisent activement, 10 gaps manquants, un plan de priorités en 7 niveaux, et un plan d'exécution en 3 cycles. Conclut avec le diagnostic en une phrase.

## Points clés

1. **Diagnostic en une phrase** : "TwoWeeks is currently closer to the better product, but it is still missing the conversion scaffolding and import trust layer needed to beat a simpler commercial funnel like ResumeLab."

2. **6 choses qui nuisent activement** : (A) Weak first-run activation — "A better system that is slower to understand often loses to a worse system that is immediately legible." (B) Import/parser trust gap — "If it can't understand my resume, I can't trust its AI." (C) AI consistency gap — "Users score AI by the worst visible output, not the average." (D) Lack of beginner path. (E) Value communication underdeveloped. (F) Risk of "beauty without reassurance."

3. **Priority 1 — Import trust** : premier à attaquer car l'import est le premier moment où l'app prouve son intelligence. Si raté : activation faible, confiance AI détruite, premium feel effondré, bounce.

4. **3 cycles d'exécution** : Cycle 1 = parser cleanup + import review layer + quick-start path + first-run checklist + preview/editor linking. Cycle 2 = AI rulebook + toolbar quality + template switcher + document health. Cycle 3 = jobs object + premium packaging + landing/value communication + "proposal" framing.

5. **Ce qui est déjà meilleur que ResumeLab** : core workspace architecture, long-term editing model, proposal/job-aware direction, visual taste, AI embedding potential, retention potential. "You are starting from a better product foundation."

6. **L'équation pour battre ResumeLab** : ResumeLab's activation discipline + TwoWeeks' superior workspace + superior design system + job-aware proposal engine + reliable import + coherent AI = category leader.

7. **Keep/Change/Avoid/Defer** : Garder skeleton editor, live preview, 3 tones, graphic design, extension. Changer l'entrée produit (pas workspace first), le pattern AI rewrite UX, l'onboarding. Éviter le modèle popup de ResumeLab, l'expansion de tones, le giant template gallery. Déférer collaboration, analytics avancés, catalog de templates géant.

## Implications pour twoweeks

- L'ordre de priorité du gap analysis **confirme et affine** la roadmap P0 existante : import recovery en tête, puis quick-start, puis AI rulebook.
- Le "Cycle 1" mappe directement sur les 4 initiatives P0 de la roadmap — c'est une validation forte.
- La notion de "conversion scaffolding" (quick-start + first-run coaching) est le gap le plus urgent à combler pour la compétition commerciale.
- Le gap analysis n'invalide aucune décision architecturale existante (skeleton editor, 3 tones, extension) — il dit juste d'ajouter une couche d'entrée en amont.
