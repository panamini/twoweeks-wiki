# Job Library PRD

## Summary

PRD for making saved jobs a first-class product object and building a lightweight Job Library behind the extension -> app -> document-generation loop.

## Key points

- Core loop: save job, understand job, generate document, keep outputs linked.
- V1 includes Job object, library page, extension save flow, Job Brief, document links, editable extracted summary.
- The surface should feel like an inbox of opportunities, not a recruiter CRM.
- Extracted job context must remain editable; no silent parsing certainty.
- V1 explicitly excludes notes, activity timeline, batch generation, batch apply, advanced ATS/CRM features, and complex statuses.

## Implications

Creates the canonical product case for [[product/job-library]] and reinforces Jobs as the next repeat-usage layer in the roadmap.
