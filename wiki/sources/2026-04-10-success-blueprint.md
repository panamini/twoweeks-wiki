# Success Blueprint — Path B

**Type** : Document de blueprint produit
**Date** : 2026-04-10

## Résumé

Blueprint complet du produit idéal TwoWeeks : thesis produit, modèle stratégique hybride funnel+éditeur, architecture en 5 objets, 3 parcours utilisateur, IA en 3 modes, pipeline d'import en 6 étapes, système de templates curé, onboarding 2026, modèle de monétisation, philosophie UI. Réponse directe à la peur fondatrice : l'éditeur peut-il convertir autant qu'un funnel ?

## Points clés

1. **Positionnement** : "The fastest way to turn your experience into premium, job-ready applications." Pas "AI resume builder", pas "beautiful templates" — mais Speed + Quality + Confidence + Repeat use.

2. **Réponse à la peur centrale** : Ne pas choisir funnel OU éditeur. Layer 1 = guided activation funnel (convertir les cold users). Layer 2 = structured editor workspace (retenir les long-term users). "The funnel creates commitment. The editor creates retention and superiority."

3. **5 objets produit** : Profile (identité core réutilisable), Experience Library (bullet bank + achievement bank par rôle), Job Record (objet normalisé : title, company, keywords, tone cues, must-haves), Resume Variant (output spécifique : template + ordering + bullets), Cover Letter/Proposal Variant (doc ciblé job + version history).

4. **3 parcours utilisateur** : Quick conversion path (cold traffic → first draft → paywall), Workspace path (retour + edit + export), Job-import path via extension (LinkedIn → tailored draft → export). Le troisième est "extremely strong if polished."

5. **IA en 3 modes** : Draft AI (guardrails forts + job awareness + anti-generic), Assist AI (rewrite/shorten/strengthen/fix/align dans la floating toolbar), Suggestion AI (2-3 variants curated, ranked, distinct — jamais 10 options). Default = 1 strong rewrite + optional "show alternatives" (max 2-3).

6. **Pipeline import idéal en 6 étapes** : Raw ingest → Structural normalization → Confidence scoring → Review queue (blocs incertains surfacés, pas silencieusement placés) → Cleanup AI (glyphes, merges, duplicates) → User confirmation. "Never dump uncertain parse results directly into final sections without review."

7. **Onboarding 2026** : Progressive checklist + contextual nudges + first-time spotlight (actions critiques seulement) + embedded empty-state education + dismissible coach panel optionnel. Pas de tooltip tour géant.

8. **Template strategy** : 1 recommended + 2 alternates max pendant la création. Puis controlled template switcher (6-12 curated max). Philosophie : "Templates should be a finishing decision, not an opening burden."

9. **IA interaction 5 règles** : (1) jamais demander de prompt quand un bouton suffit, (2) chaque action AI a un intent obvieux, (3) chaque output est replaceable pas destructif (replace/insert below/keep both/compare), (4) l'IA se souvient du contexte auto (job+tone+profile+section type), (5) IA la plus forte sur : summary, experience bullets, cover letter opening, CTA, ATS keyword adaptation.

10. **Proposal UX forte** : Left = job offer card + keywords + tone cues + extracted priorities. Right = generated proposal. Cette structure crée "The AI is writing from the job, not from nowhere" — excellent UX trust signal.

## Implications pour twoweeks

- Le concept de "Experience Library" comme objet distinct (bullet bank, achievement bank par rôle) est une évolution significative de l'architecture — à versionner dans cv-forge.
- Le "Job Record" comme first-class object est confirmé comme moat builder (convergence avec la roadmap P2).
- L'asymétrie AI (proposal avec guardrails vs toolbar moins curé) est un problème prioritaire — aligné avec AI Rulebook P0.
- L'extension LinkedIn est explicitement traitée comme "moat" et "killer path" — non-négociable dans la stratégie.
- La floating toolbar est validée comme asset fort — à renforcer, pas repenser.
- Le modèle de monétisation recommandé : free draft + pay pour export/unlimited edits/advanced AI/multi-doc management. Vendre le "productivity layer", pas juste l'export.
